# 报告规范

## 必选结构

`【沃伦与查理】`

- 由沃伦先开场：
  - 公司是做什么的
  - 生意是不是简单清晰
  - 护城河来自哪里
  - 增长引擎是什么
  - 是否值得持续跟踪
- 再由查理打断：
  - 沃伦忽略了什么
  - 这家公司未来可能怎么死
  - 三个最致命风险
  - 一个退出或警报信号
- 允许沃伦短回应一次，但不能把风险抹平。

`【本杰明插话：估值】`

- 采用插话式对话，不写成独立论文：
  - 先说明估值口径
  - 再说核心假设
  - 然后给保守 / 中性 / 乐观三种情景
  - 给未来 5 年现金流简表
  - 给终值、EV、每股合理价值区间
- 如果数据不完整，要在对话里明确“我现在用什么替代口径”。

`【综合结论】`

- 最好仍保留角色收束：
  - 沃伦一句，回答是不是好公司
  - 查理一句，回答最大的保留意见
  - 本杰明一句，回答当前价格更接近贵 / 合理 / 低估

## 写作要求

- 默认先完成 `外部 tushare-data -> deepmoat-local-data`，再写本报告。
- 先讲业务，再讲数字，再讲价格。
- 对话必须服务分析，不能写成表演。
- 没有证据的判断必须标明是推断。
- 对 A 股默认重点检查行业周期、政策扰动、现金流质量、资本开支、应收和存货变化、大客户依赖、管理层资本配置。
- 估值必须带数字，不能只写“高估/低估”。
- 每个角色都要有辨识度，但不要堆口头禅。
