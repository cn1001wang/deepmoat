# 报告规范

## 必选结构

`【投资裁决】`

- 放在最前面，先给读者一个像机构报告首页那样的提炼。
- 它不是完整报告，但也不限制在 5-8 行。
- 优先回答：
  - 这是不是好公司
  - 现在是不是好价格
  - 当前更接近买入 / 跟踪 / 回避哪一类
  - 未来 6-12 个月最关键的 2-3 个变量
  - 最大风险是什么
- 风格要求：
  - 高密度判断
  - 先结论，后简要依据
  - 语言要像“先把结论交给投资人”

`【结构化报告】`

- 这是主体部分。
- 默认按 [structured-report.md](structured-report.md) 的 12 个模块组织。
- 每个模块都应有结论，不要只堆数据。
- 对不适用的方法或资料缺口，要明确说明原因。

`【沃伦、查理与投资者问答】`

- 默认写成股东大会问答体，至少包含 3 个问题，优先 4-6 个问题。
- 每个问题都按这个节奏推进：
  - `投资者：……？`
  - `沃伦：……`
  - `查理：……`
- 允许沃伦短回应一次，但不能把风险抹平。
- 问题优先覆盖：
  - 公司到底怎么赚钱
  - 护城河和同行差异
  - 最可能怎么出问题
  - 现在是否值得继续跟踪或出手

`【本杰明插话：估值】`

- 默认也写成问答体，不写成独立论文：
  - `投资者：现在这个价格到底算贵还是便宜？`
  - `本杰明：……`
  - 先说明估值口径
  - 再说核心假设
  - 然后给保守 / 中性 / 乐观三种情景
  - 给未来 5 年现金流简表
  - 给终值、EV、每股合理价值区间
- 如果数据不完整，要在对话里明确“我现在用什么替代口径”。

`【综合结论】`

- 最好仍保留角色收束：
  - 沃伦一句，回答是不是好公司
  - 查理一句，回答最大的保留意见
  - 本杰明一句，回答当前价格更接近贵 / 合理 / 低估

## 写作要求

- 默认先完成 `外部 tushare-data -> deepmoat-local-data`，再写本报告。
- 先给投资裁决，再写 12 模块结构化报告，再进入对话。
- 对话必须服务分析，不能写成表演。
- 问答体是默认，不是可选点缀。除非用户明确不要，否则不要再写成长篇轮流发言体。
- 没有证据的判断必须标明是推断。
- 对 A 股默认重点检查行业周期、政策扰动、现金流质量、资本开支、应收和存货变化、大客户依赖、管理层资本配置。
- 估值必须带数字，不能只写“高估/低估”。
- 每个角色都要有辨识度，但不要堆口头禅。
