# analysis

当用户要做公司研究、护城河判断、风险审计、估值测算或投研写作时使用本 skill。
默认输出采用“投资裁决 + 12模块结构化报告 + 股东大会问答体 + 本杰明插话”的中文结构。

## 仓库内直达路径

在 `deepmoat` 仓库里做 A 股单公司分析时，优先走最短链路：

1. 先看 `app/models/models.py`，并记住真实表名是 `income`、`balancesheet`、`cashflow`、`daily_basic`，不是 `balance_sheet`、`cash_flow`。
2. 若只是生成一份对话体中文报告，优先复用或补丁 `scripts/analysis_dialogue_report.py`，不要每次从零写分析脚本。
3. 外部公共资料只补 2-4 条高价值来源：
   - 最新年报/半年报/三季报或业绩解读
   - 公司官网投资者关系材料
   - 1 条行业/宏观权威来源（如国家统计局、农业农村部、行业协会）
4. 不要为同一份公告反复搜索多个 PDF 镜像；拿到公司官网或交易所可验证来源后就停止扩搜。
5. 最终报告仍落到 `outputs/`，文件名使用 `{ts_code}-{name}-{date}.md`。

## 先读什么

1. 从 `app/models/models.py` 确认表结构；不要跳过这一步。
2. 阅读 [references/combo-workflow.md](references/combo-workflow.md)。
4. 公司分析默认必须先调用外部 `tushare-data`。
5. 然后必须调用 `skills/deepmoat-local-data/SKILL.md`。
6. 最后才进入本 skill 的对话式输出。

## 数据原则

- 核心数据至少合并 `income`、`balancesheet`、`cashflow`、`daily_basic`。
- 需要质量指标时补充 `fina_indicator`、`fina_audit`、`dividend`。
- 输出分析时要明确数据来源、统计口径和缺口。
- 最终报告始终使用中文输出。
- 角色化表达只改变写法，不改变证据标准；所有判断仍要落在财务和业务事实上。
- 默认优先写成“股东大会现场问答体”：投资者在台下提问，沃伦先答，查理追问或拆解，本杰明在估值问题上校准数字。
- 数据调用顺序是硬规则，不是建议：`外部 tushare-data -> deepmoat-local-data -> analysis`。
- 报告输出到outputs/目录下，命名格式是 `{ts_code}-{name}-{date}.md`，例如 `300274-阳光电源-20230101.md`。
- 如果本地库已覆盖到最新正式报告期，而外部 `tushare-data` 只用于校验行情、行业和宏观背景，就不要重复抓整套财务表。
- 若本地库时点晚于公开网页摘要，财务判断以本地库为主，同时在报告中明确“价格日期”和“财报日期”。
- 做估值时，先核对字段单位，尤其是 `total_mv`、`total_share`、`netdebt`、`fcfe`、`fcff`，不要默认再乘一次 `1e8`。


## 多市场适配

### 美股（NYSE/NASDAQ）
- SEC文件：10-K、10-Q、8-K、DEF 14A（代理声明）
- GAAP + Non-GAAP调节（关注调整项的合理性）
- 电话会记录（Earnings Call Transcript）
- 分析师共识：买方/卖方研报、目标价分布
- 期权隐含波动率参考（反映市场预期波动）
- 内部人交易（Form 4）、机构持仓变动（13F）

### 港股（HKEX）
- 港交所披露易公告
- HKFRS/IFRS会计准则
- 南向资金（大陆买港股）/ 北向资金流向
- AH溢价分析（同一公司A股vs H股的价差）
- 中资/外资持仓变动
- 联交所卖空数据

### A股
- 巨潮资讯/交易所公告
- 中国会计准则（CAS）
- 北向资金（沪港通/深港通）+ 融资融券余额
- 股东户数变化（筹码集中度指标）
- 龙虎榜（异动个股机构席位）/ 大宗交易
- 限售股解禁日历

## 重要免责声明

在每份报告末尾务必包含：

> ⚠️ **免责声明**：本分析仅供教育和研究用途，不构成投资建议。"巴菲特"视角是一个受沃伦·巴菲特公开表述的投资原则启发的分析框架——它并不代表沃伦·巴菲特本人对任何具体公司的观点。所有投资都有风险，请在做出投资决策前咨询合格的财务顾问。

## 工作流

1. 先用 `tushare-data` 拿最新公共数据：行情、财务、行业、宏观、公告、新闻。
2. 在本仓库内，优先用最小外部集补齐结论，不要把外部搜索做成全文档爬取。
3. 再用 `deepmoat-local-data` 对齐本地库，优先复用现成服务或 `scripts/analysis_dialogue_report.py`。
4. 先写一个“投资裁决”，把结论、价格判断、关键变量和风险提炼出来，按 12 个模块输出结构化报告。
5. 结构化报告完成后，再进入“股东大会问答体”。
6. 对话以投资者提问驱动，而不是三人连续自说自话。
7. 默认每个关键问题都按“投资者提问 -> 沃伦回答 -> 查理补刀/反问”的顺序推进。
8. 在估值问题上插入本杰明，专门负责口径、假设和数字纪律。
9. 结尾回到三人简短收束，给出“好公司”和“好价格”是否同时成立。
10. 对话里要区分事实、计算结果和推断，不要把角色腔调写成虚构表演。

## 降本规则

- 用户只要“分析一家公司”时，不默认展开所有可选数据源；先交付高信号版本。
- 同一个回合里，优先复用已经验证可运行的脚本和 SQL 口径，不重复造轮子。
- 如果本地库已经有最新季报，不再额外抓取同口径 Tushare 财务表，只补市场价、行业背景和管理层公开表态。
- 外部搜索命中公司官网、交易所或国家统计局后，优先停止继续检索重复来源。
- 发现数据冲突时，直接在报告里点明，不要为了“找最终正确版本”无限扩搜。
