# Value Report Skill

按以下流程为指定股票生成“可复核、可追踪、可比较”的基本面报告。

## 硬性要求
- 报告语言必须为中文。
- 涉及“最新/当前/今日”时，必须使用绝对日期。
- 财务数据优先使用本地库，字段定义以 `app/models/models.py` 为准。
- 至少覆盖近3-5年年度数据；季度数据用于补充趋势拐点。
- 关键判断必须给出数据或来源支撑；无法确认时明确写“数据缺失/待核实”。
- 必须回答“终极三问”。

---

## 执行流程（必须按顺序）

0. 先生成草稿（推荐）  
- 运行：`uv run python scripts/value_report_scaffold.py <股票代码或名称>`  
- 作用：自动生成 `outputs/value-reports/*-value-report-draft-YYYYMMDD.md`，预填财务与估值定量部分，再补充定性分析。  
- 脚本默认会追加核心图表 `option`（收入、净利、利润率、产品/地区结构、资产负债、现金流、股东回报、财务比率、ROE/ROA）。

1. 标的识别  
- 优先解析用户给出的股票代码；若只给名称，先在 `stock_basic` 映射 `ts_code`。
- 输出公司全称、代码、行业、上市日期、分析时点。

2. 数据采集（本地优先）  
- 核心表：`income`、`balancesheet`、`cashflow`、`fina_indicator`、`daily_basic`、`dividend`、`fina_audit`、`stock_company`。  
- 必算指标：
  - 成长：营收/归母净利同比、3-5年 CAGR
  - 盈利：毛利率、净利率、ROE、ROIC（可得时）
  - 健康：资产负债率、流动比率、有息负债、现金
  - 现金流：经营现金流、自由现金流、经营现金流/净利润
  - 估值：PE/PB/PS/股息率；可得时补 EV/EBITDA、PEG

3. 外部增量信息（必须补充）  
- 从公司公告（巨潮/交易所/公司官网）补最新年报、季报、分红、回购、管理层变动。  
- 从权威行业来源补行业景气与竞争格局（协会/统计局/头部研究机构）。  
- 外部资料只做“增量验证”，不替代本地财务主口径。

4. 结构化分析（按模板逐节输出）  
- 每节末尾必须有“结论”。

5. 最终判断与评分  
- 输出多头逻辑、空头逻辑、核心跟踪指标。  
- 输出总评分（100分）并给出分项权重说明。  
- 回答终极三问。

## 输出骨架（固定）

### 1. 公司概况（商业模式优先）
- 公司如何赚钱
- 收入来源构成与核心业务占比
- 客户类型（To B / To C / 政府）
- 持续性收入特征（一次性 vs 复购）
- 客户或区域集中度
结论：
- 商业模式是否简单可理解
- 是否具备长期可持续性

### 2. 行业与竞争格局
- 市场空间、阶段、增速
- 主要竞争对手
- 市场份额与行业集中度
- 公司产业链位置
结论：
- 是否优质赛道
- 公司竞争位置
- 未来3-5年趋势

### 3. 护城河分析（含真伪辨别）
- 品牌/成本/网络效应/转换成本/技术/渠道
- 真伪辨别：
  - 提价5%是否流失客户
  - 客户价格敏感度
  - 是否存在“非它不可”场景
  - 替代品难度
  - 更换供应商成本
结论：
- 护城河类型
- 强度（强/中/弱/伪）
- 是否具备真实定价权

### 4. 管理层与资本配置
- 管理层背景与稳定性
- 诚信风险（处罚/诉讼/财务治理）
- 战略理性
- 资本配置：分红、回购注销、并购、多元化、资本开支
结论：
- 管理层类型（价值创造者/中性/价值毁灭者）
- 是否值得长期信任

### 5. 财务分析

#### 5.1 成长性
- 营收与净利润近3-5年增长及稳定性  
结论：是否具备持续成长能力

#### 5.2 盈利能力
- 毛利率、净利率、ROE/ROIC  
结论：是否具备定价权、盈利质量如何

#### 5.3 财务健康
- 资产负债率、有息负债、现金储备、短债覆盖  
结论：是否存在财务风险

#### 5.4 现金流质量
- 经营现金流、自由现金流、净利润匹配度  
结论：利润是否真实、是否具备造血能力

### 6. 成长驱动
- 未来3-5年增长来源（提价/扩张/新品）
- 增长逻辑清晰度  
结论：成长是否可持续

### 7. 风险分析（含幸存者偏差）
- 政策、竞争、技术替代、财务、客户集中
- 幸存者偏差：
  - 行业历史最差阶段
  - 公司当期盈利/现金流/偿债表现
  - 极端环境下是变强、持平或衰退  
结论：抗风险能力（强/中/弱），是否穿越周期

### 8. 估值分析
- PE/PB/PS/PEG/EV-EBITDA（可得即给）
- 当前估值 vs 历史估值（可得窗口内）
- 当前估值 vs 同业对比  
结论：高估/低估/合理，是否具备安全边际

### 9. 投资判断
- 多头逻辑（3条）
- 空头逻辑（3条）
- 核心跟踪指标（3条）

### 10. 最终结论
- 是否好公司
- 是否具备长期投资价值
- 当前价格是否值得买入
- 建议：买入/观察/回避

### 11. 总评分（100分）
- 商业模式
- 护城河
- 管理层
- 财务
- 风险
- 估值
- 最终评分（/100）

### 12. 三个终极问题（必须回答）
1. 提价5%，客户会不会流失？  
2. 公司赚的钱有没有被管理层浪费？  
3. 在行业最差年份，公司是怎么活下来的？

## 输出规范
- 默认输出 Markdown 报告；用户明确要求时再附 ECharts `option`。
- 涉及金额统一标注单位（亿元/万元/元）；同比统一 `%`。
- 需要引用来源时，优先给官方链接（公告/交易所/公司官网/统计机构）。
- 结论必须区分“事实”与“推断”。

## 失败兜底
- 若本地库缺关键表：先给可得分析，再列出缺失项与影响范围。
- 若外部资料检索失败：保留本地定量结论，并明确“外部验证不足”。
- 若同一指标多口径冲突：默认采用“本地库主口径 + 公告文字解释”并说明差异原因。
